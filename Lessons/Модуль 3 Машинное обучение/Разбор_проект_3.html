Разбор проекта 3 Restaurant Rating prediction¶

В этом соревновании вам будет предложен датасет, содержащий сведения о ресторанах. С помощью имеющего в вашем распоряжении кода, вам необходимо создать модель, использующую алгоритм RandomForestRegression, которая будет прогнозировать рейтинг ресторана по версии TripAdvidor.

Вот по каким критериям проект будет оценивать ментор:¶
  • Качество кода (ясность, соответствие стандартам оформления pep-8, наличие комментариев).
  • Наличие визуализации с выводами (корреляция и важность признаков).
  • Feature Engineering и обогащение датасета (*Дополнительно: использование внешних источников данных).
  • Работа с метрикой МАЕ.

А чего нам долго ждать обучения? 

Обучим прямо сейчас и посмотрим, какой скор выходит.

# Импортируем необходимые библиотеки:
from sklearn.ensemble import RandomForestRegressor # инструмент для создания и обучения модели
from sklearn import metrics # инструменты для оценки точности модели

def ML(df):
    y = df.Rating.values # наш таргет
    X = df.drop(['Rating'], axis=1)
    
    # Воспользуемся специальной функцие train_test_split для разбивки тестовых данных
    # выделим 10% данных на валидацию (параметр test_size)
    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.1, random_state=RANDOM_SEED)
    
    # Создаём модель (НАСТРОЙКИ НЕ ТРОГАЕМ)
    model = RandomForestRegressor(n_estimators=100, verbose=1, n_jobs=-1, random_state=RANDOM_SEED)
    
    # Обучаем модель на тестовом наборе данных
    model.fit(X_train, y_train)
    
    # Используем обученную модель для предсказания рейтинга ресторанов в тестовой выборке.
    # Предсказанные значения записываем в переменную y_pred
    y_pred = model.predict(X_test)
    
    # Сравниваем предсказанные значения (y_pred) с реальными (y_test), и смотрим насколько они в среднем отличаются
    # Метрика называется Mean Absolute Error (MAE) и показывает среднее отклонение предсказанных значений от фактических.
    print('MAE:', metrics.mean_absolute_error(y_test, y_pred))
    
    # в RandomForestRegressor есть возможность вывести самые важные признаки для модели
    plt.rcParams['figure.figsize'] = (5,5)
    feat_importances = pd.Series(model.feature_importances_, index=X.columns)
    feat_importances.nlargest(15).plot(kind='barh')
    plt.show()

1 Насколько точны прогнозы модели?¶

Что такое метрика и зачем она нужна? Метрика помогает оценить качество работы вашей МЛ модели.

Вам придется подбирать метрики самостоятельно, исходя из условия вашей задачи, вашего датасета и тд.

[1] Хорошая статья на Хабре про метрики в общем и как их выбрать

[2] Еще одна хорошая статья

[3] Хороший математический сайт о машинном обучении

[4] Чем MAE отличается от RMSE на Medium

Текст взят с сайта [2]

  • Классификация (classification): У вас есть ограниченное количество ответов - метки класса (class label). Ваша модель относит данные к какому-либо классу. Например болен-не болен.

    • Confusion Matrix
    • Accuracy Metric
    • Recall/Sensitivity Metric
    • Precision Metric
    • F1 score

  • Регрессия (regression): Предсказывает какое-либо число или числовой вектор (например, завтра будет такая-то температура)

    • Mean Absolute Error (MAE)
    • Mean Squared Error (MSE)
    • Root Mean Squared Error (RMSE)

  • Ранжирование (learning to rank): Модель будет предсказывать порядок элементов. Например, нам дали учебный класс и мы должны проранжировать учеников по росту, то есть, упорядочить их от самого высокого, до самого низкого.

    • Простая метрика
    • Kendall’s tau

Важно читать про каждую метрику и понимать ее недостатки. Например, Accuracy Metric нельзя использовать при несбалансированных классах!

В вашем задании вам предложена метрика MAE: Mean Absolute Error

$$MAE = \frac{1}{n} \sum{|y - \hat{y}|} $$

-среднее абсолютное значение отклонения предсказанных значений от фактических, где $y$ - реальное значение переменной, $\hat{y}$ - предсказанное значение вашей моделью

2 Переводим категориальные значения в числовые¶

Под категориальным значением мы будем понимать признак, значения которого означают принадлежность объекта к одной или нескольким категориям. Например, марка автомобиля (одна категория), цвет или цвета (одна или несколько категорий).

  • Фиктивная переменная (англ. dummy variable) — качественная переменная, принимающая значения 0 и 1, включаемая в эконометрическую модель для учёта влияния качественных признаков и событий на объясняемую переменную. Почитать поподробнее
  • LabelEncoder -- Например, у нас есть колонка, в которой содержится ограниченный набор строковых параметров (кошечки, собачки, попугайчики). Каждому параметру мы сопоставляем число: 1 - кошечки, 2 - собачки, 3 - попугайчики. Таким образом, наша колонка содержит уже не строковые переменные, а числа (1,2,3).

ссылка на функцию

  • One-Hot Encoding - Кодирование категориальных объектов в виде одномерного числового массива.

Пример:

У нас есть словарь слов (кошечки, собачки, попугайчики).

Мы берем каждое слово и сопостовляем этому слову вектор (т.е. как бы кодируем его):

  • кошечки - [1 0 0]
  • собачки - [0 1 0]
  • попугайчики - [0 0 1]

Внимание! Длина вектора = длине словаря (Поэтому если у вас 1млн уникальных слов, то лучше OHE не пользоваться :) )
